# 首页三列与编排器入口页收口设计

## 背景

GraphiteUI 刚完成 `/` 与 `/workspace` 的合并，当前 `/` 已经成为唯一主入口。

但新的入口仍有两个明显问题：

1. 首页目前只有两列：
   - 最近运行
   - 最近图

   这不足以把“模板”这个真实入口能力抬到与运行、图同等级的位置。

2. `/editor` 入口页仍带有历史阶段痕迹：
   - 顶部文案仍是“新编排器入口”
   - 顶部按钮仍写“返回工作台”
   - 页面内部又包了一层 `max-w-7xl px-6 py-8`
   - 与外层 `LayoutShell` 的内边距叠加后，视觉上出现错位
   - 模板区与已有图区当前并非真正同权

## 目标

本轮收口三个内容：

1. 首页改成三列等权入口：
   - 最近运行记录
   - 模板选择
   - 最近图

2. `/editor` 改成真正的“模板与图并列同权”的全量入口页

3. 左侧边栏支持完全收起，并保留一个轻量展开按钮

## 核心决策

### 1. 首页改成三列，而不是保留两列

首页不再只承担“最近工作回到哪里”的入口职责，也要把“从模板开始”抬到同一层级。

因此首页主体采用三列结构：

- 左列：最近运行记录
- 中列：模板选择
- 右列：最近图

这三列是同权关系，不做主次压缩。

### 2. 中列模板只展示少量正式模板

首页中列不是全量模板目录页，而是“快速起步区”。

因此只展示少量模板卡片，数量以 3 到 5 个为宜。  
结合当前仓库现状，首页可直接展示现有正式模板。

中列底部保留一个 `更多模板` 按钮，跳转到 `/editor`。

### 3. `/editor` 作为全量入口页

`/editor` 不再表达“新阶段入口”或“临时 landing page”。

它的定位是：

“查看全部模板，并继续编辑已有图”

因此 `/editor` 页面结构应收口为两列并列：

- 左列：全量模板
- 右列：已有图

两列宽度同权，不再使用“窄模板栏 + 宽图列表”的旧比例。

### 4. 左侧边栏折叠采用“完全收起 + 悬浮展开按钮”

不采用 rail 模式，不保留缩略导航。

原因：

- 当前项目没有一套稳定的侧栏图标体系
- 强行压成 rail 会引入新的视觉语言和新的图标决策
- 当前视觉体系更适合“完整展开”与“完全收起”两态切换

因此折叠行为定义为：

- 展开态：保持现有完整侧栏
- 收起态：侧栏完全隐藏
- 左上角或页面边缘保留一个轻量展开按钮
- 折叠状态写入 `localStorage`

## 页面结构

### 首页 `/`

顶部仍然保留当前最小前厅区：

- 简短定位说明
- `新建图`
- `打开编排器`

主体改成三列：

- 最近运行记录
- 模板选择
- 最近图

列内规则：

- 最近运行：展示 5 条
- 模板选择：展示 3 个正式模板
- 最近图：展示 5 条

每列右上角保留轻量跳转：

- 最近运行：`查看运行记录`
- 模板选择：`更多模板`
- 最近图：`打开编排器`

移动端时三列纵向堆叠，顺序保持：

1. 最近运行
2. 模板选择
3. 最近图

### 编排器入口页 `/editor`

页面顶部保留一个较短的说明头部，但不再使用历史阶段性表述。

建议：

- 标题：更直接，例如“从模板开始，或继续已有图”
- 次级说明：简短说明这是全量模板与图入口，不再提“第一阶段”“新编排器入口”等历史语境
- 按钮：
  - `新建空白图`
  - `返回首页`

主体区改为真正等权两列：

- 左列：全部模板
- 右列：已有图

不再使用 `320px + 其余` 的非对称布局。

## 视觉与交互原则

- 保持当前暖色、纸张感、工作台气质
- 不新增图标库依赖
- 折叠按钮使用当前项目内部已有的轻量 SVG 风格
- 不改变编辑器主界面的布局逻辑
- 只修正 `/editor` 入口页和整体壳层的边距/层级问题

## 侧栏折叠行为

### 桌面端

- 默认展开
- 用户点击折叠按钮后，侧栏完全收起
- 主内容区自动扩展到全宽
- 页面边缘保留一个小型展开按钮
- 状态持久化到 `localStorage`

### 移动端

移动端仍可沿用同一布尔状态逻辑，但本轮不单独做复杂抽屉系统，只保证：

- 收起后不挡住主内容
- 展开按钮可见且可点击

## 非目标

本轮不处理以下内容：

- 编辑器画布页本身的布局重构
- 模板搜索、筛选、分类系统
- 侧栏 rail 图标模式
- 新增动画体系

## 验证标准

1. 首页主体为 3 列，分别展示最近运行、模板选择、最近图
2. 首页中列有 `更多模板`，跳转到 `/editor`
3. `/editor` 顶部不再出现“返回工作台”或“新编排器入口”等历史文案
4. `/editor` 主体为模板与已有图并列同权
5. 左侧边栏可以完全收起并再次展开
6. 折叠前后主内容区不会出现明显错位
7. 保持当前视觉风格，不引入新的明显设计语言
