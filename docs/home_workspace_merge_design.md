# 首页与工作台合并设计

## 背景

GraphiteUI 是一个开源的 LangGraph 可视化编排工具，既面向已经熟悉系统的专业用户，也面向第一次接触项目的新用户。

当前同时存在两个入口：

- `/`：偏介绍型首页
- `/workspace`：偏工作台总览

这两个页面在职责上已经明显重叠：

- 都在承担“产品入口”职责
- 都在解释系统能力
- 都在引导进入 editor / runs / knowledge

结果是：

- 对老用户来说，入口重复，降低日常效率
- 对新用户来说，介绍内容和真实工作内容被拆散，第一屏并不够直接

## 目标

把 `/` 和 `/workspace` 合并为一个统一入口页面，使其同时满足两类用户：

- 新用户：能快速理解这个工具是什么、从哪里开始
- 老用户：打开页面就能看到最近工作，并立即回到真实上下文

## 核心决策

### 1. `/` 作为唯一主入口

`/` 不再是独立的介绍页，也不做纯营销式首页。

它的定位是：

“带最小 onboarding 的工作台前厅”

也就是说：

- 保留最少量的产品定位说明
- 第一职责是进入真实工作内容

### 2. `/workspace` 并入 `/`

`/workspace` 不再维护独立的信息架构。

建议最终行为：

- `/workspace` 直接重定向到 `/`

这样可以避免两套入口长期并存，继续制造重复心智。

### 3. 首页主体改成真实数据双栏

首页主内容不再是能力说明卡片堆叠，而是直接展示真实工作内容：

- 左栏：最近运行
- 右栏：最近图

这是最符合工作台工具日常使用习惯的结构。

## 页面结构

### 顶部前厅区

顶部只保留一个极短的前厅区，不再使用大面积介绍模块。

包含内容：

- 一条简短定位说明
- 一个主操作：`新建图`
- 一个次操作：`打开编排器`

说明文案只回答一件事：

“GraphiteUI 是一个把 LangGraph 工作流做成可编辑、可运行、可回看的界面的工具。”

不再保留以下内容：

- 多张能力快照卡
- 多段产品说明文案
- 再次解释 workspace / editor / runtime 的流程卡

### 主内容区

主内容区采用两栏布局：

- 左栏：最近运行
- 右栏：最近图

每栏展示 5 到 6 条即可，重点是“可立即进入”，不是“完整列表页替代”。

每条记录保持现有卡片语义：

- 最近运行：状态、关联 graph、进入详情
- 最近图：名称、节点数、边数、进入编辑器

### 空态

空态必须区分新人场景，而不是只显示“暂无数据”。

建议：

- 没有 graph 时：引导创建第一个 graph
- 没有 run 时：提示先进入 editor 运行一个 graph

空态仍维持当前视觉气质，不做营销式插画或大段 onboarding。

## 信息层级

合并后，页面的信息优先级应固定为：

1. 立即进入工作
2. 查看最近上下文
3. 最小必要的产品理解

不能再回到“先介绍产品，再进入工作”的旧结构。

## 导航与路由

### 导航

侧栏中不应再同时出现两个含义相近的入口。

建议收口方向：

- 只保留一个主入口项
- 入口名称优先使用 `首页` 或 `工作台` 其中一个

建议使用：

- 中文：`首页`
- 英文：`Home`

原因：

- 路由已经是 `/`
- 对新人更自然
- 同时不影响它承担工作台前厅角色

### 路由行为

建议最终形态：

- `/`：真实主入口
- `/workspace`：重定向到 `/`

## 与现有页面的关系

合并后，原 `workspace` 页面中的以下能力需要迁移到 `/`：

- metric card
- 最近 graph 列表
- 最近 run 列表
- 快捷入口

但迁移时不应机械照搬整个 `workspace` 页面。

需要主动删减：

- 过长的标题解释
- 重复说明文案
- 为说明而说明的模块

## 实现原则

- 保持当前暖色、纸张感、工作台式视觉气质
- 不引入新的营销化视觉语言
- 不增加新的复杂动效
- 不改编辑器布局
- 优先复用现有数据获取逻辑和卡片组件

## 分步实施建议

### Phase A

先把 `/` 改造成新的统一主入口：

- 删除当前介绍型首页结构
- 引入工作台真实数据
- 保留最小前厅区

### Phase B

收口导航与路由：

- 处理侧栏入口命名
- 让 `/workspace` 重定向到 `/`

### Phase C

清理遗留：

- 删除不再使用的首页文案 key
- 删除不再使用的 workspace 独立结构
- 补充空态与回归测试

## 验证标准

完成后应满足以下标准：

1. 新用户打开 `/`，能在几秒内理解“这是什么工具”以及“从哪里开始”
2. 老用户打开 `/`，第一屏就能看到最近运行和最近图
3. `/workspace` 不再作为独立入口存在
4. 首页不再出现重复解释产品能力的模块
5. 页面保持现有视觉语言，不引入明显的新风格

## 非目标

本次改动不处理以下内容：

- 编辑器主布局重构
- 运行详情页重构
- 新的 onboarding 系统
- 面向外部官网的 marketing landing page
