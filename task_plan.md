## Key Questions

1. 第一轮前端优化是优先动首页/工作台，还是先动编辑器细节？
2. 哪些现有页面是“产品入口页”，哪些是“工作台页”，应该有明显不同的层级？
3. 在不改掉当前视觉气质的前提下，哪些微交互最值得补？

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| 保留当前暖色、纸张感、工作台式气质 | 现有产品已有辨识度，不应推倒重做 |
| 技能建议只用于辅助，不直接照搬设计系统 | `ui-ux-pro-max` 更偏 SaaS/营销页，不适合直接覆盖编辑器 |
| 先建立文件式规划 | 后续前端优化是多阶段任务，适合用持久文件跟踪 |
| 第一轮先动首页、工作台与导航壳层 | 风险较低，也最能快速提升产品入口感 |
| `/` 作为唯一主入口，`/workspace` 并入 `/` | 开源工具既要照顾新人理解，也要兼顾老用户日常效率 |
| 首页主体收口为“最近运行 + 最近图”双栏 | 更贴近日常使用路径，减少说明页和工作台页重复 |
| 首页继续升级为三列等权入口 | 模板选择也应成为首页主入口，而不是隐藏到 `/editor` 之后 |
| `/editor` 改成模板与图并列同权的目录页 | 首页承接快速开始，`/editor` 承接完整模板与图目录 |
| 侧栏折叠采用“完全收起 + 展开按钮” | 不引入 rail 图标体系，更符合当前视觉语境 |
