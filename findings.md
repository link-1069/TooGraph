# Findings & Decisions

## Requirements

- 保持 GraphiteUI 当前的整体视觉气质，不做大换皮
- 在现有基础上增强产品感、信息层级和入口清晰度
- 编辑器优化时不能破坏已有的节点布局与主要交互
- 后续前端优化应以项目内文件持续记录，避免上下文丢失

## Research Findings

- `ui-ux-pro-max` 给 GraphiteUI 的设计系统建议偏向 SaaS 转化页与微交互增强，适合首页/营销层，不适合直接覆盖编辑器主界面
- 当前前端的强项是已有统一气质：
  - 暖色纸张感
  - 左侧工作台式导航
  - 编辑器作为高密度工作区
- 当前前端的问题主要在于：
  - 首页和工作台的信息层级偏平
  - 产品入口感不足
  - 侧栏导航还比较功能性
  - 编辑器局部配置区层级还能更清楚
- 侧栏适合增强“当前上下文”和“主控台感”，但不适合重做成复杂信息面板
- `using-superpowers` 不是仓库文档替代物，它只约束工作流程和技能使用方式
- `planning-with-files` 适合替代“执行中的阶段/计划文档”，不天然替代面向仓库的稳定待办说明
- 目前 `docs/current_engineering_backlog.md` 和 `task_plan.md` 存在一定重叠：
  - 前者更像仓库级、长期待办
  - 后者更像当前会话/当前工作流的执行计划
- `docs/README.md` 当前只起索引作用，在 `docs/` 只剩一两份文档时价值较低

## Technical Decisions

| Decision | Rationale |
|----------|-----------|
| 不照搬技能产出的 `teal + orange` 全局方案 | 会冲掉现有 GraphiteUI 视觉个性 |
| 保留当前整体视觉语言，只做产品化增强 | 风险低，能持续迭代 |
| 后续优化优先级先看首页/工作台，再看编辑器细节 | 首页与工作台最能提升“产品感”，且改动风险更低 |
| `planning-with-files` 替代执行期阶段文档 | 更适合多阶段实施、持续更新和恢复上下文 |
| 导航侧栏只做层级增强，不加复杂功能 | 保持工作台壳层简洁，避免侵入编辑器主区 |
| `/` 作为唯一入口，`/workspace` 退化为重定向 | 避免双入口长期并存，减少新人和老用户的心智切换 |
| 首页只保留最小 onboarding，不再承担完整说明页角色 | 开源工具需要照顾新人，但日常使用应优先回到真实工作内容 |
| 首页主体改成左侧最近运行、右侧最近图 | 这两类数据最能承接“继续工作”和“回看结果”两种主要路径 |
| 当前验证以静态检查和路由探测为主 | `frontend/package.json` 没有测试框架，无法假装存在完整自动化覆盖 |
| 首页改成三列等权入口 | 运行、模板、图三种起点都应在首页同层出现 |
| `/editor` 保留为全量模板与已有图目录页 | 首页负责快速入口，`/editor` 负责完整浏览与进入 |
| 侧栏折叠使用完全收起模式 | 当前项目没有稳定图标体系，不适合转成 rail |

## Visual/Browser Findings

- 当前首页更像“说明页”，不是“产品入口页”
- 工作台有标题区，但整体仍偏说明性，缺少强操作聚焦
- 侧栏结构清晰，但当前上下文感和品牌主控台感还不够强
- 编辑器现有风格应保留，只适合做减噪、提层级和反馈强化
- 首页当前存在明显重复：
  - Hero 已经说明产品主链能力
  - `flow` 区再次讲 `workspace / editor / runtime`
  - 底部三卡又第三次讲 `workspace / editor / runtime`
- 首页当前空白感主要来自：
  - Hero 双列高度偏高
  - `flow` 区和底部三卡连续堆叠，造成页面过长
  - 区块间距与卡片内边距偏大
- 工作台当前也有一定重复：
  - 标题区的 `desc + metrics_desc`
  - metric 卡里的 `description`
  - `recent graphs / recent runs` 标题下的说明
  这些在信息上有交叉
- 工作台当前空白感主要来自：
  - 顶部说明区过长
  - metric 卡文案偏多
  - Quick Actions 作为整块全宽卡片，内部上下留白较大
- 本轮收口后：
  - 首页底部重复的 `workspace / editor / runtime` 三卡已删除
  - 首页区块间距与 hero 内边距已收紧
  - 工作台顶部说明、副说明和列表说明的重复文案已移除
  - 快捷入口已改成动作型文案，不再复用首页介绍语
  - 侧栏已去掉“当前所在”附加信息块，导航层级更直接
- 本轮继续推进后：
  - 首页与工作台的职责已经正式合并
  - `/` 改成最小前厅 + 最近运行/最近图双栏
  - `/workspace` 不再保留独立信息结构，改为重定向目标
  - 侧栏主导航不再保留重复的 `workspace` 入口
  - 仓库当前没有前端测试框架，因此这轮只能依靠 `tsc`、`git diff --check`、本地启动和路由探测做验证
  - 运行态验证已确认 `/workspace` 返回 `307` 并跳转到 `/`
- 本轮再次收口后：
  - 首页将进一步改为最近运行 / 模板选择 / 最近图三列
  - `/editor` 将不再使用历史阶段文案，而是模板与图并列同权的入口页
  - `/editor` 的错位问题本质上来自壳层对 `/editor` 根路由的布局归类过宽
  - 左侧边栏折叠将由壳层统一负责，并通过 `localStorage` 持久化状态
  - 运行态已确认首页输出包含“模板选择 / 更多模板”中列
  - 运行态已确认 `/editor` 输出包含“从模板开始，或继续已有图。”和“返回首页”
  - 服务端输出已确认侧栏折叠/展开按钮存在，但点击交互本轮未通过浏览器自动化工具验证
